This is a backend feature change because provider error messages and 503 response detail
are improved, Anthropic credit errors are now handled correctly, and tests are expanded.

## Plain-Language Summary

Version `v1.2.0` tightens the user-facing error messages that appear when a provider API
key is missing or when a billing/credit/rate-limit error is returned. The 503 `providers`
list now contains only the model name (e.g. `"Claude"`) rather than a full sentence, which
matches what the frontend `getApiErrorMessage` function expects. Anthropic credit-balance
errors are now caught and reported with the same short message pattern as the OpenAI quota
error. A new test covers the Anthropic credit path.

## What Changed From v1.1.0

| Area | What changed | Why |
|---|---|---|
| `analysis.py` | `unavailable_provider_messages` renamed to `unavailable_providers`; list now contains model names (`"Claude"`, `"ChatGPT"`) not full sentences | Cleaner 503 detail payload; matches frontend parsing |
| `analysis.py` | 503 `message` simplified to `"Selected AI provider unavailable."` | Shorter, consistent |
| `llm_clients.py` | `user_model_name()` helper added to return `"Claude"` / `"ChatGPT"` for user-facing messages | Avoids repeating the mapping logic |
| `llm_clients.py` | Billing error detection broadened: now catches `"credit balance"`, `"purchase credits"`, and `"billing"` in addition to `"insufficient_quota"` | Catches Anthropic low-credit errors, not just OpenAI quota errors |
| `llm_clients.py` | Billing error message shortened to `"{model_label} billing issue. Add credits or use mock mode."` | Shorter messages are safer to display directly in the frontend |
| `llm_clients.py` | Rate-limit message shortened to `"{model_label} limit reached. Try again soon."` | Same reason |
| `llm_clients.py` | Billing error `code` set to `code or "billing_error"` | Allows the original code through when available |
| `tests/test_llm_clients.py` | `AnthropicCreditError` fixture and `test_anthropic_low_credit_error_is_short_and_not_retried` test added | Ensures Anthropic credit errors are handled correctly |
| `tests/test_routes.py` | 503 test now asserts `detail["message"] == "Selected AI provider unavailable."` | Protects the updated detail contract |
